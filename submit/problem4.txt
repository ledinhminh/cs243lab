string. In the non-CS case, the MaleRecord can now point to 4 possible names,
because in AddressBook, we created 5 Records using 4 different names. Each
creation will use the code in Record class, regardless of Male or Female.
Therefore when we ignore context, the name field can have 4 possible values
depending on the calling context. Other relations suffer from the same
inprecision problem.
